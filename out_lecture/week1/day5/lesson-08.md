# 8교시: 비공식 미니 공유

## 수업 목표
- Day5에서 다룬 로컬 실행 환경 문제를 자기 경험과 연결한다.
- 부담 없는 공유를 통해 학생들이 어떤 부분에서 막혔는지 확인한다.
- Week2 Docker 수업으로 넘어가기 전에 공통 질문을 모은다.

## 운영 원칙
8교시는 공식 산출물을 요구하지 않는다. 완성도를 따지지 않고, 학생 주도형 수업으로 과하게 밀지 않으며, "내가 겪은 실행 환경 문제를 말로 정리하는 시간"으로 운영한다.

강사는 다음 태도를 유지한다.

```text
잘 만든 결과물을 고르는 시간이 아니다.
막힌 지점을 빨리 드러내는 시간이다.
질문을 많이 남길수록 Week2 수업이 쉬워진다.
```

## 진행 흐름
### 1. 개인 정리 5분
학생들은 아래 세 문장을 완성한다.

```text
오늘 가장 이해된 개념:
아직 헷갈리는 개념:
내 컴퓨터에서 실제로 겪을 것 같은 문제:
```

### 2. 짧은 공유 20분
원하는 학생이 1분 내외로 공유한다. 격식을 갖춘 발표 형식일 필요는 없다.

공유 주제 예시:

```text
설치가 제일 부담스러운 프로그램
port 충돌을 겪어 본 경험
환경변수나 설정 때문에 헷갈렸던 경험
삭제해도 남은 파일 때문에 불안했던 경험
새 컴퓨터에서 다시 실행하기 어려웠던 경험
Docker에서 제일 궁금한 점
```

강사는 공유 내용을 칠판에 범주화한다.

```text
설치
버전
port
환경변수
데이터
삭제
재현
AI 기능
```

### 3. 공통 질문 정리 15분
학생들이 낸 질문을 Week2 목차에 연결한다.

| 학생 질문 | Week2에서 다룰 위치 |
|---|---|
| 설치하지 않고 DB를 써 볼 수 있나요? | image, container |
| port가 겹치면 어떻게 하나요? | port binding |
| 데이터는 container를 지우면 사라지나요? | volume |
| 설정값은 어디에 넣나요? | env, `.env`, compose |
| 여러 프로그램을 한 번에 켤 수 있나요? | Docker Compose |
| 다 쓰고 깔끔하게 지우려면요? | cleanup |

### 4. 마무리 10분
강사는 다음 문장으로 닫는다.

```text
Week2에서 Docker를 배우는 이유는 명령어를 외우기 위해서가 아니다.
오늘 말한 불편함을 줄이기 위해서다.
```

## 학생 메모 양식
```text
내가 겪은 로컬 환경 문제:
문제가 생긴 이유 추측:
관련된 Day5 개념:
Week2에서 확인하고 싶은 것:
```

## 강사 체크리스트
- 학생들이 Docker를 "중요한 기술이라서 배운다"가 아니라 "내가 겪는 환경 문제를 줄이는 도구라서 배운다"로 이해했는지 확인한다.
- 1주차 수준에서 과한 실습 요구를 하지 않는다.
- 이해가 느린 학생에게는 용어보다 상황 설명을 먼저 묻는다.
- 8교시는 이미지 자료 없이 대화와 칠판 정리 중심으로 운영한다.
