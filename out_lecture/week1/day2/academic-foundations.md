# Week 1 Day 2 학술 근거와 교육 설계 기준

## Purpose
Day 2는 컴퓨팅 시스템 구성요소를 암기하는 시간이 아니라, 실행 중인 서비스를 관찰 가능한 증거로 설명하는 시간이다.

## Crosswalk
| Standard | Observable action | Evidence |
|---|---|---|
| CS2023 systems knowledge | compute/memory/storage/network/process를 구분한다. | component table |
| ABET-style problem analysis | 접속 실패를 network, process, file, config 원인으로 분류한다. | troubleshooting note |
| NIST NICE-style skill | 시스템 상태를 command로 확인한다. | command output |
| Bloom apply/analyze | 명령 결과를 서비스 상태 판단에 적용한다. | HTTP/process/log evidence |

## DevOps Practitioner Lens
현업에서는 "서버가 안 됩니다"라는 보고가 충분하지 않다. 어떤 host/port/protocol에서 어떤 status가 나왔고, process와 log가 무엇을 말하는지 분리해야 한다.
