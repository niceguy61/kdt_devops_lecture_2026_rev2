# 8교시: AI로 간단한 정적 웹사이트 초안 만들기

## 수업 목표
- AI Coding Tool을 사용해 아주 작은 정적 웹사이트 초안을 만든다.
- AI가 만든 결과를 그대로 믿지 않고 실행 조건, 파일 구조, 보안/비용 위험, 브라우저 확인 기록으로 검증한다.
- Day 5 발표에서 사용할 수 있도록 "무엇을 만들었고, 어떻게 확인했고, 무엇을 제외했는지"를 기록한다.

## 범위 제한
이 챌린지는 Day 5 발표 재료를 만들기 위한 별도 실습이다. 범위를 넘기면 완성도가 아니라 재현성이 무너진다.

### 결과 수정과 재확인
AI가 만든 초안은 그대로 제출물이 아니다. 학생은 최소 한 번 본문을 직접 수정하고, 그 수정이 실제 화면 또는 HTTP 응답에 반영되는지 확인한다. 예를 들어 제목, 카드 문구, 버튼 라벨, 안내 문장 중 하나를 수업 맥락에 맞게 바꾼다.

| 수정 대상 | 좋은 수정 예 | 확인 방법 |
|---|---|---|
| 제목 | 일반적인 `My Website`를 `Week 1 Local Service Guide`로 변경 | 브라우저 제목 또는 `curl` 본문 |
| 본문 설명 | AI가 만든 홍보 문구를 실행 확인 기록 중심 문장으로 변경 | 화면에서 문구 확인 |
| 버튼/링크 | 동작하지 않는 외부 링크 제거 | console error와 HTML 확인 |
| 샘플 데이터 | 실제 개인정보를 더미 데이터로 교체 | 파일 검색과 화면 확인 |

재확인 명령 예시는 다음과 같다.

```bash
curl http://localhost:8000 | grep -E 'Week 1|Local Service|DevOps'
```

`grep` 결과가 없으면 수정이 저장되지 않았거나, 서버가 다른 폴더를 제공하고 있을 수 있다. 이때는 `pwd`, `ls`, `cat index.html`을 확인한 뒤 다시 요청한다.

## 35-43분 결과 검증과 위험 제거

- 진행: 화면이 보이는지만 확인하지 않고 운영 위험을 줄인다.
- 완료 조건: 제거한 위험과 남은 한계를 기록한다.

### Visual 2: 정상 확인과 오류 확인 루프
| 확인 단계 | 질문 | evidence |
|---|---|---|
| 정상 실행 | 사이트가 로컬 서버에서 열리는가? | `curl -I http://localhost:8000` 200 |
| 수정 반영 | 사람이 바꾼 body 문구가 실제 응답에 보이는가? | `curl` 본문 또는 브라우저 화면 |
| 오류 생성 | 없는 파일 요청이 안전하게 실패하는가? | `/no-such-file.html` 404 |
| 로그 확인 | 실패 요청이 서버 로그에 남았는가? | request log excerpt |
| 정상 재확인 | 실패 뒤에도 기본 URL은 살아 있는가? | `/` 또는 `/index.html` 200 |

이 루프는 Day5 발표에서 "AI로 만들었다"보다 더 중요한 검증 흐름이다. 정상 결과와 실패 결과를 모두 확인해야 실행 조건을 설명할 수 있다.

### 검증 질문
- 새로 생긴 파일을 모두 설명할 수 있는가?
- `index.html`이 `style.css`, `app.js`를 올바른 상대 경로로 불러오는가?
- browser console에 에러가 없는가?
- secret, token, API key, real personal data가 들어 있지 않은가?
- 외부 네트워크가 없어도 열리는가?
- Day5 발표에서 3분 안에 설명 가능한가?

### 의도적 오류와 로그 확인
정상 확인 후에는 안전한 실패를 하나 만들어야 한다. 목적은 사이트를 망가뜨리는 것이 아니라, 학생이 "오류가 났을 때 어떤 기록을 확인해야 하는가"를 Day5 발표에 설명할 수 있게 하는 것이다.

가장 안전한 오류는 없는 파일 요청이다.

```bash
curl -I http://localhost:8000/no-such-file.html
```

기대 결과는 404다. 서버 터미널 로그에서 `/no-such-file.html` 요청을 찾고, 아래 표에 기록한다.

| 오류 확인 항목 | 기록 |
|---|---|
| failing command | `curl -I http://localhost:8000/no-such-file.html` |
| expected failure | 404 |
| server log excerpt | |
| 해석 | 서버는 살아 있고, 요청한 파일이 없다. |
| recheck command | `curl -I http://localhost:8000` |
| recheck result | 200 계열 상태 코드 |

학생이 JavaScript 오류까지 확인할 수 있다면 browser console에서 에러가 없는지도 본다. 단, console 확인이 어렵다면 없는 파일 404만으로도 Day3 목표는 충분하다.
